# Crimson Files — Noir RPG Companion

Односторінковий (SPA-стиль, без роутера) веб-додаток на React + Tailwind CSS для настільної рольової гри в кримінальному стилі (Tarantino/GTA noir).

## Технології

- React 19 + Vite
- Tailwind CSS 4 (через `@tailwindcss/vite`)
- `lucide-react` для іконок

## Запуск

```bash
npm install
npm run dev       # локальний dev-сервер з HMR
npm run build     # продакшн-збірка в dist/
npm run preview   # перегляд продакшн-збірки
```

## Структура

```
src/
  context/DiceContext.jsx     — глобальний стан кубика (useContext), спільний для всіх сторінок
  components/
    DiceRoller.jsx             — плаваючий (fixed) віджет кидка кубика 1-100
    CharacterPage.jsx          — універсальний "лист персонажа" (фото, досьє, ресурси, інвентар, характеристики)
    ResourceCounter.jsx        — лічильник з кнопками +/- (Здоров'я, Глузд, Патрони)
    InventoryList.jsx          — динамічний інвентар: додавання/видалення предметів (useState)
    CharacteristicsTable.jsx   — таблиця характеристик з трьома рівнями складності
    IconMap.jsx                — мапа "назва → іконка lucide"
  data/characters.js           — усі дані персонажів (ресурси, інвентар, характеристики)
  pages/
    AngelinaPage.jsx, AnnaPage.jsx, OlgaPage.jsx, YegorPage.jsx — тонкі обгортки над CharacterPage
    GalleryPage.jsx             — сторінка "Галерея" (без кубика)
  App.jsx                       — навігація між сторінками + монтування DiceProvider/DiceRoller
```

## Механіка перевірки успіху

Кидається кубик (число 1-100) — чим менше значення, тим вищий ступінь успіху.
Кожна характеристика має три пороги: **Складний** (найменший поріг, найважче досягти),
**Середній**, **Легкий** (найбільший поріг, найлегше досягти).

Перевірка на певному рівні складності успішна, якщо `поріг характеристики >= результат кубика`.

Коли гравець кидає кубик через глобальний плаваючий віджет, останній результат зберігається
в `DiceContext` і автоматично підсвічує в таблиці характеристик найкращий досягнутий рівень
успіху для кожного рядка — не потрібно рахувати вручну.

## Дані персонажів

Усі стартові значення (здоров'я, глузд, патрони, інвентар, пороги характеристик) винесені
в `src/data/characters.js` — щоб змінити цифри чи додати нового персонажа, достатньо
відредагувати цей файл і додати відповідну сторінку-обгортку в `src/pages/`.

Фото персонажів наразі — плейсхолдер (іконка). Щоб додати реальне зображення:

1. Покладіть файл, наприклад, `src/assets/angelina.jpg`.
2. У `src/data/characters.js` додайте зверху `import angelinaPhoto from '../assets/angelina.jpg';`
   і в об'єкті персонажа замініть `photoPlaceholder: null` на `photoPlaceholder: angelinaPhoto`.

Так само для решти персонажів.

## Мобільна версія

Сайт адаптивний: навігація згортається в бургер-меню, таблиця характеристик на вузьких
екранах (< 640px) перетворюється на стек карток замість таблиці (щоб колонки
"Складний / Середній / Легкий" не обрізались), а плаваючий кубик враховує safe-area
відступи (для iPhone з "чолкою"/смужкою знизу).

## Фонова анімація

`src/components/AmbientBackground.jsx` — суто декоративний нерухомий шар позаду контенту:
пульсуюча малинова віньєтка, дрейфуючі частинки пилу/іскор, що піднімаються вгору, і легкий
рух "прожектора". Лого в шапці має ефект мерехтливої неонової вивіски (`neon-flicker` в
`index.css`). Усі анімації вимикаються автоматично, якщо в системі користувача увімкнено
"Зменшити рух" (`prefers-reduced-motion`), і не впливають на клікабельність елементів
(`pointer-events-none`).
